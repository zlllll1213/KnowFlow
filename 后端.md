你现在负责开发一个名为 KnowFlow 的智能知识库系统后端。

项目定位：
KnowFlow 是一个基于 RAG 的智能知识库问答平台，支持用户注册登录、创建知识库、上传文档、文档解析任务、向量化存储、基于知识库的智能问答。

本轮开发目标：
请优先完成后端项目骨架和核心业务接口，不要一次性实现过于复杂的大模型功能。要求项目可以正常启动，并提供清晰的目录结构、接口说明和基础实现。

技术栈要求：
1. 主后端使用 Spring Boot 3。
2. 构建工具使用 Maven。
3. 数据库使用 PostgreSQL。
4. ORM 使用 MyBatis-Plus。
5. 缓存和任务队列使用 Redis。
6. 文件存储预留 MinIO 接口。
7. 鉴权使用 JWT。
8. 接口风格使用 RESTful API。
9. 所有接口统一返回 Result<T>。
10. 使用 Docker Compose 编排 PostgreSQL、Redis、MinIO。
11. 暂时不要接真实大模型 API，先预留接口和模拟返回。

请实现以下核心功能：

一、用户认证模块

接口：
1. POST /api/auth/register
2. POST /api/auth/login
3. GET /api/auth/me

要求：
- 用户注册字段：username、password、email。
- 密码必须使用 BCrypt 加密。
- 登录成功返回 JWT token。
- 后续接口通过 Authorization: Bearer token 访问。
- 用户表至少包含 id、username、password_hash、email、created_at、updated_at。

二、知识库模块

接口：
1. POST /api/kb
2. GET /api/kb/list
3. GET /api/kb/{id}
4. PUT /api/kb/{id}
5. DELETE /api/kb/{id}

要求：
- 每个知识库必须绑定 user_id。
- 用户只能访问自己的知识库。
- 字段包含 id、user_id、name、description、created_at、updated_at。
- 删除知识库时可以先做逻辑删除。

三、文档模块

接口：
1. POST /api/document/upload
2. GET /api/document/list?kbId=
3. GET /api/document/{id}
4. DELETE /api/document/{id}
5. GET /api/document/{id}/status

要求：
- 上传文档时需要传 kbId 和文件。
- 第一版可以先把文件保存到本地 storage/ 目录，同时预留 MinIOStorageService。
- 保存文档元数据到 document 表。
- 文档状态包括：UPLOADED、PARSING、EMBEDDING、DONE、FAILED。
- 上传完成后创建 parse_task 任务，并将任务 ID 推入 Redis 队列。
- 暂时不需要真正解析文档，但要保证任务创建流程完整。

四、聊天问答模块

接口：
1. POST /api/chat/session
2. GET /api/chat/session/list?kbId=
3. POST /api/chat/ask
4. GET /api/chat/history?sessionId=

要求：
- chat/ask 请求参数包括 kbId、sessionId、question。
- 需要校验用户是否有权限访问该知识库。
- 第一版可以返回模拟答案，例如：“这是基于知识库的模拟回答”。
- 保存用户问题和助手回答到 chat_message 表。
- 预留后续调用 Go RAG 服务的 RagClient。
- RagClient 中先写一个 mockAsk 方法，后续可替换为 HTTP 调用 Go 服务。

五、数据库表设计

请在 sql/init.sql 中创建以下表：
1. user_account
2. knowledge_base
3. document
4. document_chunk
5. chat_session
6. chat_message
7. parse_task

要求：
- 所有表都包含 created_at、updated_at。
- 需要合理设计主键、外键或逻辑关联字段。
- document_chunk 表预留 embedding 字段，可使用 vector 类型注释说明，第一版可以先不强制启用 pgvector。

六、Redis 任务队列

要求：
- 文档上传后创建 parse_task。
- 将 taskId 推入 Redis list，例如 key 为 knowflow:parse:queue。
- 提供 TaskService.createParseTask(documentId)。
- README 中说明后续 Python Worker 会从该队列消费任务。

七、统一返回格式

Result<T> 示例：
{
  "code": 0,
  "message": "success",
  "data": {}
}

错误示例：
{
  "code": 40001,
  "message": "Unauthorized",
  "data": null
}
