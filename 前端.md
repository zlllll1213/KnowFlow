
你现在负责开发一个名为 KnowFlow 的智能知识库系统前端。

项目定位：
KnowFlow 是一个基于 RAG 的智能知识库问答平台，用户可以注册登录、创建知识库、上传文档、查看文档解析状态，并在指定知识库中进行智能问答。

本轮开发目标：
请完成前端项目骨架和核心页面。要求页面可以正常启动，路由完整，接口调用结构清晰。后端接口如果暂时不可用，请使用 mock 数据或封装好的 API 方法占位，但不要把业务逻辑写死在页面里。

技术栈要求：
1. 使用 Vue3。
2. 使用 TypeScript。
3. 构建工具使用 Vite。
4. UI 组件库使用 Element Plus。
5. 状态管理使用 Pinia。
6. 路由使用 Vue Router。
7. HTTP 请求使用 Axios。
8. 样式可以使用 SCSS 或普通 CSS。
9. 项目整体风格要求简洁、现代、适合简历项目展示。
10. 不要使用过于花哨或难维护的动画。

请实现以下页面：

一、登录页 Login.vue

功能：
- 用户输入 username 和 password。
- 调用 POST /api/auth/login。
- 登录成功后保存 token。
- 跳转到 Dashboard。
- 页面需要简洁美观。

二、注册页 Register.vue

功能：
- 用户输入 username、email、password、confirmPassword。
- 前端校验两次密码是否一致。
- 调用 POST /api/auth/register。
- 注册成功后跳转登录页。

三、主布局 MainLayout.vue

要求：
- 顶部 Header 显示系统名称 KnowFlow、当前用户、退出登录。
- 左侧 Sidebar 显示导航菜单。
- 导航包括：
  1. 工作台
  2. 知识库
  3. 文档管理
  4. 智能问答
  5. 系统设置
- 右侧为路由内容区。

四、工作台 Dashboard.vue

展示内容：
- 知识库数量。
- 文档数量。
- 已完成解析文档数量。
- 最近会话。
- 最近上传文档。
- 可以先使用 mock 数据，但要封装成后续可替换 API。

五、知识库列表 KnowledgeBaseList.vue

功能：
- 展示当前用户的知识库列表。
- 支持新建知识库。
- 支持编辑知识库名称和描述。
- 支持删除知识库。
- 每个知识库卡片显示名称、描述、文档数量、更新时间。
- 点击卡片进入知识库详情页。

接口：
1. GET /api/kb/list
2. POST /api/kb
3. PUT /api/kb/{id}
4. DELETE /api/kb/{id}

六、知识库详情 KnowledgeBaseDetail.vue

展示内容：
- 知识库名称和描述。
- 文档列表。
- 上传文档按钮。
- 进入问答按钮。
- 文档解析状态标签。

七、文档管理 DocumentManage.vue

功能：
- 选择知识库。
- 上传 PDF、DOCX、TXT、Markdown 文件。
- 展示文档列表。
- 展示文档状态：UPLOADED、PARSING、EMBEDDING、DONE、FAILED。
- 支持删除文档。
- 支持刷新状态。

接口：
1. POST /api/document/upload
2. GET /api/document/list?kbId=
3. DELETE /api/document/{id}
4. GET /api/document/{id}/status

八、智能问答 Chat.vue

页面布局：
- 左侧：知识库选择、会话列表。
- 中间：聊天窗口。
- 右侧：引用来源面板 SourcePanel。

功能：
- 用户选择知识库。
- 创建或选择会话。
- 输入问题并发送。
- 展示用户问题和助手回答。
- 第一版可以普通 HTTP 返回答案，不要求 SSE。
- 为后续 SSE 流式输出预留代码结构。
- 右侧 SourcePanel 用于显示召回的文档片段，可以先用 mock 数据。

接口：
1. POST /api/chat/session
2. GET /api/chat/session/list?kbId=
3. POST /api/chat/ask
4. GET /api/chat/history?sessionId=

九、Axios 封装

request.ts 要求：
- 设置 baseURL，从 .env.development 中读取 VITE_API_BASE_URL。
- 请求拦截器自动携带 Authorization: Bearer token。
- 响应拦截器统一处理 Result<T>。
- 如果返回未登录状态，清除 token 并跳转登录页。

十、Pinia 状态管理

auth store：
- token
- userInfo
- login
- logout
- fetchUserInfo

kb store：
- knowledgeBaseList
- currentKb

chat store：
- currentSession
- messages

十一、路由守卫

要求：
- 未登录用户访问主页面时跳转 /login。
- 已登录用户访问 /login 时跳转 /dashboard。

3. 不要写死大量 mock 数据在页面内部，mock 数据可以放到单独文件或通过 API 层临时返回。
